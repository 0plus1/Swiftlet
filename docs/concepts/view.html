Views

Views are files that contain layout elements for pages, written in HTML. PHP code can be used but should be limited to presentational logic, all other code belongs in the accompanied Controller. View are always loaded by a Controller and never called directly.

Views are stored in the /_views/ directory.

siteName ?>

Class

Name
View
Location
/_app/View.php

Public functions

  • load ( string $file )
    Loads a View.
  • output ( )
    Output loaded views.
  • allow_html ( string $v )
    Decode an HTML entity encoded string.

Public variables

  • $rootPath
    Absolute path to the root. Note that this can be different from $controller->rootPath (e.g. when URL rewrites or AJAX is used).
  • $viewPath
    Absolute path to the View directory (/_views).
  • $siteName
    Name of the website (value of $this->app->siteName, safe for HTML).
  • $siteCopyright
    Copyright of the website (value of $this->app->siteCopyright, safe for HTML).
  • $siteDesigner
    Name of the website designer (value of $this->app->siteDesigner, safe for HTML).
  • $pageTitle
    Title of the page (value of $controller->pageTitle, safe for HTML).
  • $pageDescription
    Description of the page (value of $controller->pageDescription or $this->app->siteDescription if missing, safe for HTML).
  • $pageKeywords
    Keywords relevant to the contents of the page (value of $controller->pageKeywords or $this->app->siteKeywords if missing, safe for HTML).
  • $inAdmin
    Used to distinguish between regular and administrative pages.
