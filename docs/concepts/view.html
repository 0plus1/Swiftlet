Views

Views are files that contain layout elements for pages, written in HTML. PHP code can be used but should be limited to presentational logic, all other code belongs in the accompanied Controller. Views are always loaded by a controller and never accessed directly.

Views are stored in the /_views/ directory.

Example View
siteName ?>

The main View class handles everything that is presented to the user. This includes HTML code, URLs, localization and more.

Public variables

  • $absPath
    Absolute path to the website's root directory in the browser.
  • $action
    Shortcut to $args[1]. The first argument in the URL is generally used to specify an action, e.g. "edit" or "delete", followed by the id number of a row in the database (see Routes).
  • $args
    List of arguments passed in the URL (see Routes).
  • $controller
    Name of the controller that is being called.
  • $id
    Shortcut to $args[2], generally used in combination with $action (see Routes).
  • $inAdmin
    Used to distinguish between regular and administrative pages.
  • $siteName
    Name of the website (see Configuration).
  • $siteCopyright
    Copyright of the website (see Configuration).
  • $siteDesigner
    Name of the website designer (see Configuration).
  • $pageDescription
    Description of the page (see Configuration).
  • $pageKeywords
    Keywords relevant to the contents of the page (see Configuration).
  • $pageTitle
    Title of the page (see Controller).
  • $path
    Path to the current controller (see Routes).
  • $request
    Path to the current controller, including arguments (see Routes).
  • $rootPath
    Path to the website's root directory in the browser.
  • $viewPath
    Path to the views directory in the browser (/_views/).
