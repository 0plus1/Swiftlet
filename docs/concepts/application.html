The Application

The Application is the core of the Swiftlet framework.

In plugins, controllers and views the Application can be accessed through the $this->app object, an instance of the Application class. Many global functions and variables are stored here (e.g. $this->app->config).

Public functions

  • check_dependencies ( array $dependencies )
    Use to make sure a required plugin is installed and ready (e.g. $this->app->check_dependencies(array('db')) if database access is needed). If not, an error message is displayed.
  • clear_cache ( )
    Calls the clear_cache hook to clear caches.
  • email ( string $to, string $subject, string $body [ , array $headers ] )
    Send an e-mail. Requires an E-mail plugin.
  • end ( )
    Calls the end hook and stops the script.
  • error ( string $errNo, string $errStr [, string $errFile ] [, string $errLine ] [, array $errCont ] )
    Use to display an error message and stop the script.
  • hook ( string $hook )
    Use to create a hook.
  • hook_register ( string $plugin, array $hooks )
    Used in plugins to register hooks.

Public variables

  • $config
    Configuration values, see Configuration.
  • $configMissing
    This variable is set to TRUE if the configuration file is missing.
  • $controller
    Instance of the controller, see Controllers.
  • $debugOutput
    Debug notices generated throughout the code. The contents are shown when an error occurs (and send to the console) if config['debugMode'] is set to TRUE in /_config.php. The notices can be used to track down bugs.
  • $userIp
    The user's internet IP address (this can be IPv4 or IPv6).
  • $view
    Instance of the view, see Views.

Instances of plugins are also available as public variables, e.g. $this->app->db (see Plugins).

Constants

  • VERSION
    Application version number.
