Views

Creating a View

Views (stored in /_views/) contain layout elements for pages, mostly HTML, images and Javascript. PHP code can be used but should be limited to presentational logic (all other code should go in the Controller).

Example View
siteName ?>

Class

$view

Public variables

  • $view->rootPath
    Absolute path to the root. Note that this can be different from $contr->rootPath (e.g. when URL rewrites or AJAX are used).
  • $view->viewPath
    Absolute path to the View directory (/_view).
  • $view->siteName
    Name of the website (value of $model->siteName, safe for HTML).
  • $view->siteCopyright
    Copyright of the website (value of $model->siteCopyright, safe for HTML).
  • $siteDesigner
    Name of the website designer (value of $model->siteDesigner, safe for HTML).
  • $view->pageTitle
    Title of the page (value of $contr->pageTitle, safe for HTML).
  • $view->pageDescription
    Description of the page (value of $contr->pageDescription or $model->siteDescription if missing, safe for HTML).
  • $view->pageKeywords
    Keywords relevant to the contents of the page (value of $contr->pageKeywords or $model->siteKeywords if missing, safe for HTML).
  • $view->inAdmin
    If set to TRUE the current page is an administration page.

Public functions

  • $view->load ( string $file )
    Used to load a View (an HTML file in the /view/ directory).
