Auth token

Forms can not be processed if the auth_token field is missing (a security measure to prevent cross-site request forgery). Always include the following line when creating forms that are send as POST:

Confirmation

Information send as GET does not have an auth token to ensure its integrity and can't be trusted. When using GET to perform adminstrative tasks (like deleting a page using a link as ?id=1&action=delete), confirmation should be requested from the user using the $model->confirm() function (see Model).

Example
form->validate(array(
	));

if ( isset($model->GET_raw['action']) && $model->GET_raw['action'] == 'delete' )
{
	if ( !$model->POST_valid['confirm'] )
	{
		$model->confirm('Are you sure you wish to delete this page?');
	}
	else
	{
		// Delete the page
	}
}
?>

The $model->confirm() function exits the code and displays a form to the user with an option to proceed or cancel. If the user chooses to proceed the page is reloaded and $model->POST_valid['confirm'] is set to TRUE (this requires the Form plug-in).

Handling user input

For security reasons $_GET and $_POST can not be used. It is better to use the filtered altenatives below.

Safe for HTML
  • $model->GET_html_safe
  • $model->POST_html_safe
Safe for database insertion (requires a database plug-in)
  • $model->GET_db_safe
  • $model->POST_db_safe
Unfiltered (and unsafe!)
  • $model->GET_raw
  • $model->POST_raw
